HOW TO USE THESE PROMPTS
These 9 prompt parts are designed to be fed sequentially into any capable AI coding assistant. Follow this
workflow for the best results:
Step 1
Choose your AI tool
Best options: Claude Code (terminal), Cursor IDE, or ChatGPT-4o. Claude Code is the strongest for
Java projects — it writes files directly to disk. For ChatGPT/Claude.ai: copy-paste each part and ask
it to write the file.
Step 2
Start a fresh conversation
Always start a new chat/session before feeding Part 1. Don't mix this project with other
conversations — context leakage causes bugs.
Step 3
Feed Part 1 first
Say exactly this before pasting: 'You are building a Java desktop application. Here is the full
specification:' then paste Part 1. Wait for the AI to acknowledge before continuing.
Step 4
Feed parts in order
Parts 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9. For each part say: 'Continue building. Here is the next
part:' then paste. Don't skip parts — each builds on the previous.
Step 5
Test after Part 4
After Part 4 (Infrastructure), stop and test. Run Main.java. Make sure the app opens, the DB
connects, and the config loads. Fix errors before feeding Part 5.
Step 6
Save every generated file
As the AI generates each class, immediately save it to the correct path. Use the filename comment
at the top of each file: // FILE: src/main/java/...
Step 7
Handle errors
If the AI stops mid-file: say 'Continue from where you left off'. If there's a compilation error: paste the
error message and say 'Fix this error'. If it hallucinates a method: paste the error and the relevant
Part section.
Step 8
Final test
After all parts: mvn clean package. Run the JAR. Test: sign up as student → pick course →
generate AI quiz → answer → see AI review. This flow validates Parts 1–7 all work together.
